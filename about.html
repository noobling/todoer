Code Architecture

CSS can get messy quite fast that is why so many languages are built on top of it to make it more maintainable. e.g. SASS, LESS, stylus.

I hav minimised page specific styles by adopting this approach by Shay Howe

In my opinion this makes our code much more maintainable and reduces code repetition. Even though it may seem more complicated initially it reduces our technical debt

Extra Effort

Use of es6 code Most of the code is in es5 because I felt that, that was what was required in this unit. However es6 is the future, it improves of many issues people had with es5 e.g. block-level scopes, cleaner syntax, classes, loops. Much like what HTML5 was to HTML 4.01

Fake persistance with local storage Sometimes it may seem like your data is persisted between pages. This is done through storing some of your data in localstorage. I haven't done it for every page because it is time consuming and messy to attempt it in vanilla es5.

Reponsiveness through flex box and grids reponsiveness now is a requirement not an option for modern websites.
